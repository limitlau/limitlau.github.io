    • MySQL函数

    概述：在MySQL中，为了提高代码的重用性及隐藏实现细节，MySQL就提供了许多的函数，函数也可以理解为别人封装好的代码

    分类：

    聚合函数

    概述：用来统计，既称之为聚合函数，也叫统计函数或者组函数

    分类：

    COUNT 计算个数

    SUM 求和

    MAX 最大值

    MIN 最小值

    AVG 平均值

    拓展：group_concat() 用户实现行的合并，先根据group by指定列进行分组，并且用分隔符进行分隔，将同一个分组中的值进行连接，并且返回一个字符串结果

    格式：group_concat([distinct] 字段名 [order by 排序字段] [separator ‘分隔符’])

    1.使用distinct可以排除重复值

    2.如果需要对结果进行值排序可以使用order by

    3.separator是一个字符串的值，默认是’,’

    数学函数

    1.round(x,[保留位数]) 四舍五入

    • 当对正数进行四舍五入：按照正常的计算方式，四舍五入即可
    • 当对负数进行四舍五入：先把符号丢一边，对去掉负号后的正数进行四舍五入，完成以后，再把这个负号补上即可

    2.向上取整：ceil(x)

    窗口函数

    分类：

    分布函数：percent_rank() cume_dist()

    前后函数：lag(expr,n) lead(expr,n)

    序号函数：row_number() rank() dense_rank()

    头尾函数：first_value(expr) 返回第一个值 last_value(expr) 返回最后一个值

    其他函数：nth_value(expr,n) 返回第n个expr值 nile(n) 将有序的数据分为n个值，记录等级数

    开窗聚合函数：MAX MIN SUM AVG

    视图

    概述：视图（view）是一个虚拟表，非真实存在，其本质是根据SQL语句获取动态的数据集，并为其命名，用户使用时只需使用视图名称即可获取结果集，并可以将其当作表来使用。数据库中只存放了视图的定义，并没有存放视图的数据。

    使用视图查询数据时，数据库系统会从原来的表中取出对应的数据。因此，视图中的数据是依赖于原来表中的数据的。一旦表中的数据发生变化，显示在视图中的数据也会发生改变。

    作用：

    • 简单化代码，将重复代码进行使用并且封装，同时可以使复杂的查询易于理解和使用
    • 安全，如果一张表有很多数据，很多信息不需要别人看到，此时就可以使用视图

    语法：

    create [or replace] [algorithm ={undefined | merge | temptable}]

    view 视图名[(column_list)]

    as 查询语句

    [with [cascaded | loacl ] check option]

    修改视图：

    alter view 视图名 as 修改之后的查询语句

    关于试图数据更新：

    可以使用update delete insert ，但是试图如果包含下面的任何一种，那么都是不可操作的：聚合函数、distinct、group by、having、union、join、where字句中的子查询、引用from子句中的表

    视图的重命名：

    rename table 视图名 to 新的视图名；

    删除视图：

    drop view if exists 视图名

    存储过程

    存储过程是什么？

    就是一组数据库语句集合，是可以实现比较复杂的逻辑功能，和java语言中方法类似。

    存储过程也是数据库SQL代码层面的代码封装及重用

    存储过程是什么时候开始的？

    存储过程并不是MYSQL出来之后就存在的，而是在MYSQL5.0版本开始之后才支持的。

    存储过程的特性：

    是有输入及输出的参数，也是可以声明变量，并且是有控制流程语句（if…else case while…)存在

    函数的普通特性：模块化，代码的复用，封装

    速度快，只有首次执行时需要经过编译和优化，后面的调用可以直接执行，而不需要执行上面的步骤

    事务的基本操作：

    1、开启事务 Start Transaction

    命令：begin 或 Start Transaction

    2、提交事务 Commit Transaction

    成功之后

    commit

    3、回滚事务 Rollback Transaction

    失败之后

    rollback

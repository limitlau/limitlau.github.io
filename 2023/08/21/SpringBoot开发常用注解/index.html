org.springframework.boot

@SpringBootConfiguration

应用程序应该只包含一个@SpringBootConfiguration且一般都使用@SpringBootApplication使之包含在其中

与Configuration等效，且有助于在集成测试中进行配置检测

  • @AliasFor(annotation = Configuration.class)
    boolean proxyBeanMethods() default true;
    是否代理@Bean方法

org.springframework.boot.autoconfigure.

@SpringBootApplication

标识一个声明了一个或多个@Bean方法的配置类，同时触发自动配置和组件扫描。

效果等同于@SpringBootConfiguration + @EnableAutoConfiguration + @ComponentScan

  • @AliasFor(annotation = EnableAutoConfiguration.class)
    Class<?>[] exclude() default {};
    排除指定的自动配置类，使其不再生效

  • @AliasFor(annotation = EnableAutoConfiguration.class)
    String[] excludeName() default {};
    排除指定类名对应的自动配置类，使其不再生效

  • @AliasFor(annotation = ComponentScan.class, attribute = “basePackages”)
    String[] scanBasePackages() default {};
    扫描指定的包名中带注释的组件

  • @AliasFor(annotation = ComponentScan.class, attribute = “basePackageClasses”)
    Class<?>[] scanBasePackageClasses() default {};
    扫描指定的类所在的包下所有带注释的Component

  • @AliasFor(annotation = ComponentScan.class, attribute = “nameGenerator”)
    Class<? extends BeanNameGenerator> nameGenerator() default BeanNameGenerator.class;
    指定给Spring容器中检测到的组件命名的类

  • @AliasFor(annotation = Configuration.class)
    boolean proxyBeanMethods() default true;
    是否代理@Bean方法

@EnableAutoConfiguration

开启Spring应用程序上下文的自动配置，尝试猜测和配置可能需要的bean。

  • String ENABLED_OVERRIDE_PROPERTY = “spring.boot.enableautoconfiguration”;
    开启自动配置时用于覆盖的环境属性

  • Class<?>[] exclude() default {};
    排除指定的自动配置类，使其不再生效

  • String[] excludeName() default {};
    排除指定类名对应的自动配置类，使其不再生效

org.springframework.context.annotation.

@ComponentScan

  • @AliasFor(“basePackages”)
    String[] value() default {};
    不需要其他属性时，可不标明属性名进行简写
  • @AliasFor(“value”)
    String[] basePackages() default {};
    扫描指定的包名中带注释的组件
  • Class<?>[] basePackageClasses() default {};
    扫描指定的类所在的包下所有带注释的Component
  • Class<? extends BeanNameGenerator> nameGenerator() default BeanNameGenerator.class;
    指定给Spring容器中检测到的组件命名的类
  • Class<? extends ScopeMetadataResolver> scopeResolver() default AnnotationScopeMetadataResolver.class;
    指定用于解析检测到的组件的范围的类
  • ScopedProxyMode scopedProxy() default ScopedProxyMode.DEFAULT;
    指示是否为检测到的组件生成代理，默认值遵循用于执行实际扫描的组件扫描程序的默认行为
    设置此属性将覆盖为scopeResolver设置的任何值
  • String resourcePattern() default ClassPathScanningCandidateComponentProvider.DEFAULT_RESOURCE_PATTERN;
    控制符合组件检测条件的类文件
    考虑使用更灵活的方法如includeFilters和excludeFilters
  • boolean useDefaultFilters() default true;
    指示是否对带有@Component、@Repository、@Service、@Controller注解的类的自动检测
  • Filter[] includeFilters() default {};
    指定进行组件扫描的类型
    进一步将扫描范围缩小到basePackages中能匹配给定过滤器的
    指定过滤器会匹配指定的basePackages中任何类型，即使不被默认过滤器匹配（没有@Component注解的类）
  • Filter[] excludeFilters() default {};
    指定不进行组件扫描的类型
  • boolean lazyInit() default false;
    指定是否将注册扫描的bean进行延迟初始化
  • @interface Filter
    • FilterType type() default FilterType.ANNOTATION;
      过滤器的类型
    • @AliasFor(“classes”)
      Class<?>[] value() default {};
    • @AliasFor(“value”)
      Class<?>[] classes() default {};
      用作过滤器的类，根据type属性配置的值来指定类，有多个类时使用or逻辑
      实现了BeanFactoryAware、BeanClassLoaderAware等Aware接口的自定义类型过滤器中的方法会在匹配前被调用
    • String[] pattern() default {};
      用于过滤器的模式，作为指定Class值的替代方案
      如果type为ASPECTJ，则为ASPECTJ类型模板表达式。type为REGEX，则为匹配完全限定名类的REGEX模式

org.springframework.transaction.annotation.

@Transactional

描述单个方法或类上的事务属性

  • @AliasFor(“transactionManager”)
    String value() default “”;

  • @AliasFor(“value”)
    String transactionManager() default “”;

    指定事务管理器

  • String[] label() default {};

    定义事务标签，用来描述事物

  • Propagation propagation() default Propagation.REQUIRED;

    事务传播类型，默认支持当前事务，不存在则创建新事物

  • Isolation isolation() default Isolation.DEFAULT;

    事务隔离级别，默认为数据库隔离级别

  • int timeout() default TransactionDefinition.TIMEOUT_DEFAULT;

    事物的超时时间，以秒为单位，默认为底层事务系统的默认超时

  • String timeoutString() default “”;

    同上，返回值类型不同

  • boolean readOnly() default false;

    若事务只读可设置为true，允许运行时进行相应优化

  • Class<? extends Throwable>[] rollbackFor() default {};

    指定须引起事务回滚的异常类型，默认会在RunTimeException和Error上回滚，推荐使用，类型安全地匹配异常类型及其子类

  • String[] rollbackForClassName() default {};

    指定须引起回滚的异常名称（必须是Throwable子类）

  • Class<? extends Throwable>[] noRollbackFor() default {};

    指定不能导致事务回滚的异常类型，首选方法，已类型安全的方式匹配异常类型及其子类

  • String[] noRollbackForClassName() default {};

    指定不能导致回滚地异常类型（必须是Throwable子类）
